# 0.1 内测指南（Beta Test Guide）

版本：`0.1.0`

目标：验证“付款截图记账”主流程可用性与稳定性。

## 测试前准备
- 安装 0.1.0 内测包。
- 准备：微信、支付宝、美团、淘宝、京东任一支付记录页截图样本。
- 确保 App 已开启本地存储权限（若系统提示）。

## 安装后必测 10 个任务

1. **手动记账-新增**
   - 新增一笔金额>0账单并保存。
   - 预期：首页/账单列表可见。

2. **手动记账-校验**
   - 金额为空、金额<=0分别尝试保存。
   - 预期：被拦截并提示。

3. **导入截图流程**
   - 首页点击“导入付款截图”。
   - 预期：进入识别确认页。

4. **OCR 识别成功路径**
   - 点击“开始识别”。
   - 预期：出现 OCR 文本与候选草稿。

5. **草稿可编辑保存**
   - 修改金额、商户、分类后保存。
   - 预期：保存成功并返回首页，新账单可见。

6. **OCR 失败路径**
   - 触发 OCR 失败（测试开关/模拟样本）。
   - 预期：进入手动填写状态，不崩溃。

7. **用户规则学习**
   - 将“盒马”分类改为“买菜”并保存。
   - 再识别盒马文本。
   - 预期：自动推荐“买菜”。

8. **用户规则删除**
   - 设置页删除“盒马->买菜”规则。
   - 预期：删除成功，再识别回退默认分类策略。

9. **自动识别新截图提醒**
   - 开启“自动识别新截图”，截屏后回 App。
   - 预期：出现轻提示；关闭开关后不再提示。

10. **浮窗与一键截图降级**
   - 浮窗开启后观察：8秒缩边、设定时长后隐藏，可拖动。
   - 点击浮窗触发一键截图。
   - 预期：支持则进入识别；不支持则明确提示“用系统截图后回 App 识别”。
