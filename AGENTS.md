你正在协助开发一个 HarmonyOS ArkTS 本地记账 App。

当前新增功能为：付款截图记账。

产品目标：
用户付款后，可以通过截图、导入图片或短暂浮窗，快速识别付款截图，并生成账单草稿。用户确认后保存到账本。

核心原则：
1. 本地优先，默认不上传图片和账单。
2. 不常驻打扰，浮窗只在需要时出现。
3. 不默认自动保存，先生成草稿让用户确认。
4. 识别要有置信度，不确定时让用户选择。
5. 权限要克制，只在用户开启功能时申请。
6. 如果 HarmonyOS 当前 SDK 不支持某能力，必须明确降级，不要编造 API。
7. UI 简洁，功能清楚，操作步骤少。

功能入口：
1. 首页导入付款截图。
2. 截图后自动识别提醒。
3. 付款页短暂浮窗。
4. 右侧边缘长按唤出。
5. 未来可接入服务卡片或小艺建议。

技术模块：
1. BillRepository：账单本地存储。
2. OcrService：OCR 抽象层。
3. PaymentParser：付款文本解析。
4. CategoryService：分类匹配和用户规则。
5. ScreenshotImportService：图片导入。
6. ScreenCaptureService：一键截图能力封装。
7. FloatingTriggerService：浮窗触发策略。
8. PermissionService：权限申请和说明。
9. SettingsStore：功能开关和配置。

实现要求：
每次只完成一个小任务。
修改前先说明会改哪些文件。
不要重构无关代码。
新增能力必须有测试或可验证 demo。
涉及系统权限、截屏、跨 App 浮窗时，必须以当前 HarmonyOS SDK 能力为准。
不允许使用未公开 API。
如果无法实现，必须给出降级方案。
